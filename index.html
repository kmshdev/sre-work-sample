Pull-request timeline pattern

The reviewer should see a coherent evidence trail.

  • C
    Candidate commits the runtime-fleet slice

    One operator surface, three simulated runtimes, and one mock external dependency are runnable from local commands.

  • CI
    Smoke validation passes from a clean checkout
    submission-report.md smoke output
    + runtime-a healthy, runtime-b paused, runtime-c degraded
    + dependency timeout detected and blocked unsafe work
    ! document expected output, not just command names
  • R
    Operational scenarios are defended

    Each implemented failure shows detection, operator decision, blocked unsafe work, recovery, and residual risk.

  • M
    Submission route is opened

    Candidate invites kmshdev as collaborator or reviewer after the repository and report are complete.
