Assignment description

Your task is to prove an operating model, not breadth.

Build a local runtime slice

Use fake data, local services, and mock integrations. Do not require paid infrastructure, real accounts, or hidden local state.

Expose operator decisions

Show which runtimes are healthy, degraded, paused, blocked, or in incident response, and which actions are unsafe.

Defend the evidence

Capture command output, logs, metrics, screenshots, workflow runs, or transcripts that prove the system behavior.

Required implementation scope

Close these gates before submitting.

Gate Requirement Evidence expected
Runtime fleet Three simulated runtimes Each runtime needs separate identity, configuration, health, and dependency boundary. Control output, health states, and runtime-specific evidence.
Mock dependency One external boundary Model a fake data feed, account service, gateway, broker, or equivalent dependency. Healthy and failed dependency behavior with operator impact.
Safety control Pause or block unsafe work Preserve state while preventing unsafe operations. Command, API, or UI evidence that unsafe work is blocked.
Failure scenarios Two implemented scenarios Include detection, state transition, alert or runbook path, recovery, and proof. Trigger commands, logs, health changes, and recovery output.
Release safety Bad rollout or bad configuration scenario Show validation, promotion, rollback, and blast-radius thinking. Automation output or a documented validation plan.

Suggested timeline

Use the 7 days to reduce risk early.

  1. Day 1

    Make the fresh checkout work.

    Install, run tests, run smoke, and decide what you will replace.

  2. Days 2-3

    Build the smallest control surface and health model.

    Add three runtimes, one mock dependency, and one pause or block path.

  3. Days 4-5

    Implement failure and release-safety evidence.

    Trigger two scenarios, recover them, and capture proof.

  4. Day 6

    Wire delivery automation and cleanup commands.

    Make validation repeatable with CI, Make, Taskfile, or equivalent.

  5. Day 7

    Write the submission and defense notes.

    Separate implemented behavior from planned future work.
