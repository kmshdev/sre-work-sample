Recommended pacing

  1. Run the starter and read the output shape.
  2. Define how liveness differs from safe-to-serve eligibility.
  3. Make stale data for one feed fail closed while the other feeds remain usable.
  4. Add or improve tests around stale, unknown, and recovered states.
  5. Wire evidence into GitHub Actions, logs, or captured command output.
  6. Write the incident note, alert rationale, rollback note, and tradeoffs.

make setup

.venv/bin/python -m sre_work_sample.cli scenario stale \
  --feed bravo \

.venv/bin/python -m sre_work_sample.cli status --state /tmp/bravo-stale.json

Hard gates

  • Fresh checkout setup works.
  • Tests and smoke command work locally.
  • GitHub Actions runs setup, tests, smoke, and docs validation.
  • A stale feed is detected and fails closed.
  • Recovery is reproducible.
  • No real accounts, paid infrastructure, or secrets are required.

Do not spend time here

  • Do not turn the assignment into a platform build.
  • Do not make cloud deployment, Kubernetes, Terraform, DNS, BGP, or edge networking required.
  • Do not build a polished dashboard unless it directly improves evidence.
