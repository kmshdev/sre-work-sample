Start here

Run the starter before changing it.

make setup
make test
make smoke
make run

Keep these commands working or replace them with equivalent commands. The reviewer must be able to clone, set up, run, smoke, and clean up from a machine with no prior local state.

Suggested 7-day path

Reduce review risk in this order.

Day 1: Fresh checkout

Make setup, tests, smoke, and cleanup repeatable before adding scope.

Days 2-3: Runtime model

Add three runtimes, one dependency, and one safety control.

Days 4-5: Failure evidence

Implement two scenarios with detection, blocking, and recovery.

Day 6: Automation

Wire validation, smoke checks, promotion thinking, and rollback notes.

Day 7: Submission

Write the report, no-agent incident, AI usage, and defense notes.

Always: Cut scope

Prefer a small system with strong evidence over a broad demo.

Deliverables

Every deliverable should answer a reviewer question.

Deliverable Question it answers
Fresh-checkout commands Can a reviewer run the system without hidden local state?
Smoke command Does the control surface, runtimes, dependency, and health work?
Failure scenarios Can the system fail visibly and recover through operator steps?
Delivery automation Can validation and release safety be repeated?
Defense notes Can you explain tradeoffs and production risks live?

Scope control

When stuck, cut scope before adding tools.

  • Keep one local control surface instead of building several interfaces.
  • Use one mock dependency and make its failure behavior clear.
  • Prefer captured command output over long prose explanations.
  • Document future work separately from implemented behavior.
  • Rerun the smoke command after each scenario change.
