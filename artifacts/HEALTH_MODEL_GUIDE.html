Explain the operating model, not just the code

What We Are Testing

  • Operational judgment under partial failure.
  • Evidence quality: can another engineer rerun the proof?
  • Safety decisions when market data is stale or unknown.
  • Restraint: solving the assignment without building a platform.

What We Are Not Testing

  • UI polish.
  • Cloud deployment.
  • Kubernetes or Terraform implementation.
  • Trading strategy.
  • Vendor familiarity.

Health Model

Expose these fields in a way the reviewer can inspect.

FieldMeaning
alive The feed consumer can answer.
last_tick_age_seconds Age of the newest tick.
freshness_status fresh, stale, or unknown.
safe_to_serve Whether unsafe downstream work may continue.
allowed_actions Actions still allowed for the feed.
blocked_reason Why unsafe work is blocked.

Fail-Closed Reasoning

Stale or unknown data should block unsafe actions. Fresh unaffected feeds should remain eligible. Recovery requires evidence that freshness returned, not only that a process restarted.

Follow-Up Interview Prep

  • Which stale-data alert pages, and which alert creates a ticket?
  • What changes when the model grows beyond three feeds?
  • How would a feed freshness threshold roll out safely?
  • Which AI suggestion did you reject or correct?
