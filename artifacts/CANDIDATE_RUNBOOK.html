Use the checklist to pace the work

This checklist is static on purpose. Capture evidence in your repository; do not rely on browser state.

Start at 0 of 8 complete

Mark progress in your own notes or submission. The page does not store state.

  1. Clone and install

    Run make setup.

    Evidence: setup transcript or no-error command output.

  2. Run the baseline

    Run make test and make smoke.

    Evidence: passing tests and smoke JSON.

  3. Trigger stale bravo

    .venv/bin/python -m sre_work_sample.cli scenario stale \
  --feed bravo \
  --output /tmp/bravo-stale.json

    Evidence: bravo appears in unsafe_feeds.

  4. Recover bravo

    .venv/bin/python -m sre_work_sample.cli recover \
  --feed bravo \
  --state /tmp/bravo-stale.json \
  --output /tmp/bravo-recovered.json

    Evidence: overall_status returns to eligible.

  5. Document the health model

    Fill docs/ARCHITECTURE.md.

    Evidence: liveness, freshness, eligibility, and blocked actions.

  6. Write operations evidence

    Fill docs/OPERATIONS.md and capture transcripts under evidence/.

    Evidence: trigger, detection, blocked action, and recovery.

  7. Complete submission files

    Fill SUBMISSION.md, INCIDENT_NOTE.md, AI_USAGE.md, and DEFENSE_NOTES.md.

    Evidence: all required links and notes are present.

  8. Run final validation

    Run make test, make smoke, and make docs-check.

    Evidence: all pass before inviting kmshdev.
